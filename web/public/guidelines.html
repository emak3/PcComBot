質問ガイドライン

質問をスムーズに解決するためのガイドラインです。必ず読んでから質問してください。

重要事項

当ガイドラインは、質問をスムーズに解決する為に制定しているものです。回答者から質問されるのは、質問の記載に不備があります。

ガイドライン順守を行っていない質問はモデレーターにより削除等の措置を行います。

検索して調べる

質問前に必ずGoogleなどで検索してください。

調べた事や調べてわかったこと・実践した事について記載してください。

役に立たなかった関連するサイトのリンクを含めることで、あなたの質問が他のものとどのように違うのかを他の人が理解するのに役立ちます。

検索のコツ

  • エラーメッセージをそのまま検索する
  • 英語での検索も試してみる
  • 複数のキーワードを組み合わせる
  • 公式ドキュメントを確認する

一目で内容がわかるタイトルを書く

フォーラム上にはあなたの質問タイトルと詳細の冒頭一部が表示され、多くのメンバーがその情報をもとに答えられるものかどうか、興味があるかどうかを判断しています。

タイトルを読んで意味が分からなかったり、興味を抱かなかったりすれば回答を得られない可能性があります。

タイトルの書き方

タイトルをどのように書いていいのかがわからない場合は、タイトルは最後に書きます。質問の詳細部分を先に書いたほうが、今起きている問題を説明しやすくなることもあります。

良いタイトル例
  • Windowsで「○○.dll」が見つからないエラー
  • Pythonで△△する方法
  • Minecraftが実行できない
  • CPUとGPUの違いについて

正しく伝わりやすい質問の本文を書く

人に何かを質問するときは、質問内容と回答に必要な情報を正しく伝える必要があります。

質問前のチェックリスト

  • 抽象的な質問ではないですか？
  • 何を教えてほしいのかはっきりしていますか？
  • 他の人が読んでもあなたが思っている通りに伝わりますか？
  • その質問に答えるために必要な情報は記載されていますか？

必要な情報

目的

目的がわからないと、他の解決手段の提案ができません。

現状

目的の解決手段として現在取り組んでいること。

悪い例

××ってどうやったらできるか

良い例

○○をしたい。そのために××をしたいのでどのようにやる方法があるか。現状は〜〜

使用しているもの

パソコンの問題の場合

  • OS: Windows 10、Ubuntu 21.04 など
  • パーツ: 問題の起きているパーツが判明しているならばそのパーツを書く
  • PC型番: 購入したPCならば型番など

プログラムの問題の場合

  • OS: Windows 10、Ubuntu 21.04 など
  • 言語: Python 3.7、Node.js 14 など
  • ライブラリ: pipやnpmなどでインストールしたライブラリ群
  • コード: シンタックスハイライトをつけること

コードの書き方

シンタックスハイライトの付け方：

```言語名(py, js, rb等)
コード
```

重要： エラーやコードはコピペで全文を貼り付けること（スクリーンショットだと回答者の環境で再現するのが手間です）

質問禁止事項

スクレイピング関連

明示的に許可されてないサイトに対するスクレイピング行為に関する質問及び回答の禁止

理由

一般的に推奨されない行為であり、明示的な禁止をしていない場合でもその他利用規約条項に抵触する行為である為。

問題が解決したら

質問者の方へ

質問者は最後に解決方法についてまとめたものを記載してからクローズしてください。（自己解決した場合も同様です）

クローズの方法

解決につながったメッセージを右クリックもしくは長押しをしてアプリからBestAnswerを選択してください。

放置されているスレッドについて

管理者およびモデレーターがスレッドのクローズを要求することがあります：

  • 解決している場合: スレッドをまとめ、クローズしてください
  • 解決していない場合: まだ解決していない旨を知らせてください

事情によりクローズしない事をモデレーターから許可された場合にも最低週1回は状況報告を行ってください。

@有識者（回答者）へ

  • ガイドライン違反の質問に回答することを禁じます

    ガイドラインに準拠した質問になるよう質問者に促してください

  • 質問者がクローズするまでクローズ扱いにしないでください
  • 優しく対応してください

    ガイドラインを守ってない投稿には優しくガイドラインのリンクを貼ってあげてください
